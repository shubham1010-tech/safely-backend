2. SCAM SIMULATION ANALYSIS API

---

ROUTE:
POST /api/analyze

PURPOSE:
Analyze the user's decision during a phishing simulation.

REQUEST BODY:
{
"choice": "click"
}

LOGIC:
IF:

* choice = "click"

RETURN:
{
"success": false,
"riskLevel": "HIGH",
"result": "❌ This was a phishing scam!",
"explanation": "The message used urgency and a suspicious verification link.",
"tips": [
"Never click suspicious links",
"Check the sender domain carefully",
"Banks never ask for verification via email",
"Look for spelling mistakes in URLs"
],
"timestamp": "..."
}

IF:

* choice = "ignore"
  OR
* choice = "report"

RETURN:
{
"success": true,
"riskLevel": "SAFE",
"result": "✅ Great decision! You avoided the scam.",
"tips": [
"Always verify suspicious messages",
"Report phishing attempts immediately"
],
"timestamp": "..."
}

====================================================

---

3. AI CHAT ASSISTANT API

---

ROUTE:
POST /api/chat

PURPOSE:
Analyze suspicious user messages using AI-style logic.

REQUEST BODY:
{
"message": "Your account is blocked. Click here immediately."
}

LOGIC:
Create a smart keyword-based scam detector using:

* urgent
* verify
* blocked
* click here
* OTP
* account suspended
* reward
* free money

If suspicious keywords detected:
Return warning response.

EXAMPLE RESPONSE:
{
"success": true,
"isScam": true,
"riskLevel": "HIGH",
"reply": "⚠️ This message appears suspicious because it creates urgency and asks you to click a link.",
"detectedKeywords": ["blocked", "click here"],
"safetyTips": [
"Do not share OTPs",
"Avoid clicking unknown URLs",
"Verify through official websites"
],
"timestamp": "..."
}

IF SAFE:
Return educational safe response.
